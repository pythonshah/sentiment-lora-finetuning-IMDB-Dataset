# Model Card: IMDB Sentiment Classifier (LoRA-tuned DistilBERT)

## Model Details

- **Base model:** `distilbert-base-uncased`
- **Fine-tuning method:** LoRA (Low-Rank Adaptation) via HuggingFace PEFT
- **Task:** Binary sentiment classification (positive / negative)
- **LoRA target modules:** `q_lin`, `v_lin` (query and value projections in attention blocks)
- **LoRA rank:** 8 (see the rank ablation study in the notebook for the r=4/8/16 comparison)
- **Trainable parameters:** a small fraction of the base model's ~67M parameters (exact
  percentage printed at training time via `model.print_trainable_parameters()`)

## Intended Use

This model classifies English-language **movie reviews** as positive or negative. It is
intended as a portfolio / learning project demonstrating parameter-efficient fine-tuning,
not as a production sentiment-analysis service.

**Reasonable uses:**
- Classifying movie or media reviews
- Educational demonstration of LoRA fine-tuning
- Baseline for further experimentation (larger base models, more LoRA targets, etc.)

**Out of scope:**
- Production content moderation or automated decision-making
- Domains far from movie reviews (see the Limitations section — out-of-distribution
  testing in the notebook shows degraded performance on unrelated domains)
- Languages other than English

## Training Data

- **Dataset:** IMDB Dataset of 50K Movie Reviews (50,000 reviews, balanced 25k positive /
  25k negative)
- **Split:** 80% train / 10% validation / 10% test, stratified by label
- **Preprocessing:** Truncated/padded to 256 tokens using the DistilBERT tokenizer

## Evaluation

See `test_metrics.json` (generated by the notebook) for exact numbers from the most recent
training run. The notebook additionally reports:
- Confusion matrix and full classification report on the held-out test set
- Comparison against a TF-IDF + Logistic Regression baseline
- A LoRA-rank ablation study (r=4, 8, 16) trading off trainable parameters vs. F1
- Out-of-distribution qualitative testing (sarcasm, negation, non-movie domains)

## Limitations

- Trained only on movie reviews; the out-of-distribution test in the notebook shows
  reduced (though not necessarily terrible) accuracy on other domains like restaurant or
  product reviews.
- Struggles with strong sarcasm and mixed-sentiment text, as shown in the error analysis
  section of the notebook.
- English only; no multilingual support.
- Trained on a specific historical snapshot of IMDB reviews (2011 dataset release) — may
  not reflect contemporary review style or vocabulary.

## How to Use

```python
from transformers import AutoModelForSequenceClassification, AutoTokenizer
import torch

model_dir = "./my_first_finetuned_model"
tokenizer = AutoTokenizer.from_pretrained(model_dir)
model = AutoModelForSequenceClassification.from_pretrained(model_dir)

text = "This movie was absolutely wonderful!"
inputs = tokenizer(text, return_tensors="pt", truncation=True, padding=True, max_length=256)
with torch.no_grad():
    probs = torch.softmax(model(**inputs).logits, dim=-1).squeeze()

print("positive" if probs[1] > probs[0] else "negative", probs.max().item())
```

Or use the included CLI: `python src/inference.py --model_dir ./my_first_finetuned_model --text "..."`,
or the Gradio demo: `python app.py --model_dir ./my_first_finetuned_model`.
